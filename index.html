Comprendiendo Git y GitHub

La base de todo trabajo en equipo.

¿Qué son y por qué son importantes?

Git es un sistema de control de versiones. Es como una "máquina del tiempo" para tu código. Te permite registrar cada cambio que haces, volver a versiones anteriores si algo sale mal y trabajar en nuevas características sin afectar el proyecto principal.

GitHub es una plataforma de alojamiento de repositorios. Es el lugar donde subes tus proyectos para que otros los vean y colaboren contigo. Piensa en GitHub como un Google Drive pero hecho para código, con funcionalidades extra para trabajo en equipo.

Git

Software local en tu computadora.

Controla versiones de forma individual.

GitHub

Servicio en la nube (web).

Permite colaborar en proyectos.

Comandos Clave con Ejemplos

git init

Inicializa un nuevo repositorio Git local en la carpeta actual.

git init

git add

Añade uno o más archivos al área de preparación para el próximo commit.

  • Añadir un archivo específico: git add index.html
  • Añadir todos los archivos nuevos o modificados: git add .
git add index.html
# o
git add .

git commit

Guarda los cambios preparados en el historial del repositorio con un mensaje descriptivo.

git commit -m "feat: añadir estructura inicial del proyecto"

git status

Muestra el estado de los archivos y los cambios pendientes en el repositorio.

git status

git push

Sube los commits locales al repositorio remoto (GitHub).

git push origin main

git pull

Descarga los cambios del repositorio remoto al repositorio local.

git pull origin main

¿Qué es un Commit?

Un **commit** es una instantánea de tu proyecto en un momento dado. Cada vez que haces un commit, Git toma una "foto" de tus archivos en ese estado y la almacena en tu historial. Piénsalo como guardar el progreso en un videojuego.

Un buen mensaje de commit es clave. Debe explicar de forma clara, concisa y en menos de 50 caracteres qué cambio se hizo. Por ejemplo: "feat: añadir navegación principal" o "fix: corregir error de estilo".
